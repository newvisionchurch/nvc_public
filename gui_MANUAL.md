## Part 1 — 관리자(Admin) 매뉴얼

> 이 섹션은 **관리자 1명만** 진행합니다.  
> 팀원 계정 추가 및 권한 설정은 관리자가 수행합니다.

---

### 1-1. 최초 설치

#### 1단계 — 저장소 확인

```powershell
cd C:\Projects\nvc_network
```

이미 파일이 있다면 이 단계를 건너뜁니다.

#### 2단계 — 의존성 설치

```powershell
pip install -r gui\requirements.txt
```

---

### 1-2. Admin 계정 초기 생성

> **최초 1회만** 실행합니다. 이미 `gui/config/users.json`이 있다면 건너뜁니다.

```powershell
python gui\init_users.py
```

아래와 같이 순서대로 입력합니다.

```
=== NVC Network GUI — 초기 Admin 계정 설정 ===

Admin 계정 ID: admin
표시 이름 (예: Chris Jeong): Chris Jeong
비밀번호 (8자 이상): ********
비밀번호 확인: ********

Admin 계정 'admin' 생성 완료.
저장 위치: C:\Projects\nvc_network\gui\config\users.json
```

> **주의:** `gui/config/users.json`은 git에서 제외됩니다. 비밀번호 해시가 저장되므로 외부 유출에 주의하세요.

---

### 1-3. SSH 접속 설정

`gui/config/ssh_targets.example.json`을 복사하여 `ssh_targets.json`을 만들고 실제 정보를 입력합니다.

```powershell
copy gui\config\ssh_targets.example.json gui\config\ssh_targets.json
```

```json
{
  "nas":  { "host": "NAS_IP", "username": "nas_user", "port": 22 },
  "efg":  { "host": "EFG_IP", "username": "root",     "port": 22 },
  "ap_defaults": {
    "username": "ubnt",
    "port": 22,
    "reboot_command": "reboot"
  }
}
```

> 비밀번호는 파일에 저장하지 않습니다. 첫 실행 시 입력하면 Windows Credential Manager에 저장할지 묻습니다.

---

### 1-4. GUI 실행

```powershell
# VPN을 먼저 연결한 뒤 실행
python gui\main.py
```

또는 실행 스크립트:

```powershell
.\gui\run-gui.ps1
```

GUI는 항상 **최대화 상태**로 시작됩니다.

---

### 1-5. Admin 첫 로그인

#### 단계 1 — ID / 비밀번호 입력

- **ID**: 설정한 admin 계정 ID 입력
- **비밀번호**: 설정한 비밀번호 입력
- **로그인** 버튼 클릭

#### 단계 2 — TOTP 2단계 인증 초기 설정 (최초 1회)

첫 로그인 시 QR 코드 화면이 표시됩니다.

1. 스마트폰에서 **Google Authenticator** 또는 **Authy** 앱을 엽니다.
2. **+** 버튼 → **QR 코드 스캔**을 선택합니다.
3. 화면의 QR 코드를 스캔합니다.
4. 앱에 나타난 **6자리 코드**를 입력란에 입력합니다.
5. **설정 완료** 버튼을 클릭합니다.

> QR 코드 스캔 후 앱에 "NVC Network GUI (계정ID)" 항목이 추가됩니다.  
> 이후 로그인마다 앱의 6자리 코드(30초마다 갱신)를 입력합니다.

#### 단계 3 — 메인 화면 진입

인증 성공 시 메인 화면으로 진입합니다.  
Admin 계정은 **사용자 관리** 탭이 표시됩니다.

---

### 1-6. 팀원 계정 추가

메인 화면 → **사용자 관리** 탭 → **새 사용자 추가** 버튼

| 항목 | 설명 |
|------|------|
| 계정 ID | 팀원이 로그인할 ID (영문, 숫자) |
| 표시 이름 | GUI에 표시될 이름 (예: 홍길동) |
| 비밀번호 | 초기 비밀번호 8자 이상 — 팀원에게 별도 전달 |
| 비밀번호 확인 | 동일하게 재입력 |
| 역할 | `admin` / `operator` / `viewer` 중 선택 |
| 권한 | 아래 [권한 구조 참조표](#권한-구조-참조표) 참고 |

**저장** 버튼을 클릭하면 계정이 생성됩니다.

> 팀원에게 **계정 ID**와 **초기 비밀번호**를 안전한 방법으로 전달하세요.  
> TOTP QR 코드는 팀원이 첫 로그인 시 직접 스캔합니다.

---

### 1-7. 팀원 계정 편집

**사용자 관리** 탭 → 해당 팀원 선택 → **편집** 버튼

변경 가능 항목:
- 표시 이름
- 역할 (admin / operator / viewer)
- 활성 계정 체크 (해제 시 로그인 불가)
- 권한 개별 설정
- **비밀번호 변경...** 버튼으로 비밀번호 초기화

---

### 1-8. TOTP 초기화 (팀원 인증 앱 분실 또는 기기 변경 시)

**사용자 관리** 탭 → 해당 팀원 선택 → **TOTP 초기화** 버튼

> 초기화 후 해당 팀원이 다음 로그인 시 새 QR 코드를 다시 스캔하게 됩니다.

---

### 1-9. 팀원 계정 삭제

**사용자 관리** 탭 → 해당 팀원 선택 → **삭제** 버튼

> 현재 로그인된 본인 계정은 삭제할 수 없습니다.

---

### 1-10. 개발자 모드 등록 (관리자 PC 자동 로그인)

등록된 PC에서는 비밀번호·TOTP 없이 자동으로 로그인됩니다.

1. GUI 실행 → 정상 로그인 (1회)
2. **사용자 관리** 탭 → 본인 계정 선택 → **이 컴퓨터 개발자 등록** 버튼
3. 다음 실행부터 자동 로그인, 상단에 `⚙ 개발자 모드` 배지 표시

> Machine ID는 Windows 레지스트리 MachineGuid 기반으로 VPN 연결 여부와 무관하게 안정적입니다.

---

### 1-11. 일상 실행 명령 (관리자)

```powershell
# VPN 연결 후
python gui\main.py
```

## Part 3 — 탭별 기능 상세

권한에 따라 일부 탭이나 버튼이 비활성화될 수 있습니다.

---

### Log 동기화 탭

> `can_sync_nas` 권한이 있는 사용자만 사용 가능합니다.

NAS에 저장된 Logstash JSONL 로그를 PC 로컬 캐시로 동기화합니다.  
PC에 없는 파일 또는 크기가 다른 파일만 다운로드합니다 (차분 동기화).

**실행 방법:**
1. **NAS Log 동기화** 버튼을 클릭합니다.
2. 비밀번호 입력창이 표시되면 NAS 비밀번호를 입력합니다.  
   (이전에 저장한 경우 창이 표시되지 않습니다.)
3. 동기화가 완료되면 상태 메시지가 업데이트됩니다.

동기화된 로그는 `runtime/logs/ap/` 에 저장됩니다.

---

### AP ELK Log 탭

ELK 로그(로컬 캐시 JSONL) 기반으로 AP별 Stuck 이벤트를 집계합니다.  
실제 로그 타임스탬프 기반이므로 Stuck 발생 여부의 **ground truth**입니다.

**ELK Log Stuck Count 실행 방법:**
1. 분석 기간을 선택합니다 (날짜 피커).
2. **ELK Log Stuck Count** 버튼을 클릭합니다.
3. AP별 Stuck 횟수와 색상 상태가 표시됩니다.

**색상 기준:**
| 횟수 | 상태 |
|------|------|
| 0 | Healthy (초록) |
| 1~2 | Warning (노랑) |
| 3~9 | Suspect (주황) |
| 10+ | Critical (빨강) |

**AP 상세 분석:**
1. AP 목록에서 분석할 AP를 클릭하여 선택합니다.
2. **상세 보기** 버튼을 클릭합니다.
3. 팝업 창에 Stuck 유형별 로그 샘플이 표시됩니다.

**찾기:** AP 이름 또는 ID로 목록을 필터링합니다.

---

### AP Remote 탭

AP에 직접 SSH 접속(EFG relay 경유)하여 `mca-dump` 와 `logread` 를 실행하고  
무선 하드웨어 상태 진단 정보를 표시합니다.

> 조회는 EFG → sshpass → AP 경로로 이루어집니다. AP 비밀번호와 EFG 비밀번호가 필요합니다.  
> 비밀번호는 최초 입력 후 Windows Credential Manager에 저장할 수 있습니다.

#### 전체 조회 / 선택 조회

- **전체 조회**: 인벤토리의 모든 AP(29대)를 동시 조회 (최대 4개 병렬)
- **선택 조회**: 목록에서 선택한 AP만 조회 (Shift로 다중 선택 가능)
- **중지**: 진행 중인 조회를 중단합니다

#### 진단 컬럼 설명

| 컬럼 | 의미 | 임계값 초과 시 |
|------|------|--------------|
| **ELK Stuck** | ELK Log 기반 누적 Stuck 이벤트 수 | `⚠` 표시 |
| **로그오류** | 현재 부팅 세션 중 stuck/ath_reset/vap timeout 패턴 수 (logread grep) | `⚠` 표시 |
| **재시작2G/5G** | 드라이버 레벨 무선 채널 리셋 횟수 (`ast_ath_reset`) | `⚠` 표시 |
| **VAP지연2G/5G** | VAP 초기화 타임아웃 횟수 (`timeout_waiting_for_vap_cnt`) | `⚠` 표시 |
| **CU2G%/CU5G%** | 채널 이용률 (`cu_total`) — 높을수록 채널 혼잡 | `⚠` 표시 |
| **2.4G Cli/5G Cli** | 현재 연결된 클라이언트 수 | — |
| **2.4G Ch/5G Ch** | 현재 사용 중인 무선 채널 | — |
| **CPU%/Mem%** | AP CPU 및 메모리 사용률 | — |
| **Uptime** | 마지막 재부팅 이후 경과 시간 | — |
| **응답(ms)** | SSH 응답 왕복 시간 (mca-dump 포함) | — |

> **ELK Stuck** 컬럼은 AP ELK Log 탭에서 **ELK Log Stuck Count** 를 실행한 뒤에만 표시됩니다.  
> 로그오류 = 0 이어도 ELK Stuck > 0 이면 과거에 Stuck이 발생했을 수 있습니다 (부팅 세션 차이).

#### 컬럼 정렬

모든 컬럼 헤더를 클릭하면 오름차순/내림차순으로 정렬됩니다.  
정렬된 컬럼 헤더에는 `↑` 또는 `↓` 가 표시됩니다.

#### 상세 보기

AP를 선택(Shift로 최대 3개 다중 선택 가능)하고 **상세 보기** 버튼을 클릭하면  
각 AP의 mca-dump 원본 데이터가 별도 창으로 표시됩니다.  
3개를 초과하여 선택한 경우 처음 3개만 표시됩니다.

#### ⚠ 설정 — 임계값 변경

**⚠ 설정** 버튼을 클릭하면 각 컬럼의 경고 임계값을 변경할 수 있습니다.  
변경 후 **저장** 버튼을 누르면 `app_config.json`에 저장되어 다음 실행 시에도 유지됩니다.

기본 임계값:
| 컬럼 | 기본값 | 의미 |
|------|--------|------|
| ELK Stuck | 2 | 초과 시 경고 |
| 재시작2G/5G | 100 | 초과 시 경고 |
| VAP지연2G/5G | 20 | 초과 시 경고 |
| CU2G%/CU5G% | 70 | 초과 시 경고 |
| 로그오류 | 0 | 1 이상이면 경고 |

#### AP Reset — 다중 선택 후 일괄 실행

> `can_reset_ap` 권한이 있는 사용자만 사용 가능합니다.

하단 버튼 배치:
```
[상세 보기]    [AP Reset 선택] [AP 선택 해제] | [모두 선택] [모두 해제] ·· [AP Reset 실행]
```

**Reset 대상 지정 방법:**

| 버튼 | 동작 |
|------|------|
| **AP Reset 선택** | 테이블에서 클릭한 AP를 빨간색으로 Reset 대상 등록 (Shift 다중 선택 지원) |
| **AP 선택 해제** | 클릭한 AP를 Reset 대상에서 해제 (Shift 다중 선택 지원) |
| **모두 선택** | 전체 AP를 Reset 대상으로 지정 |
| **모두 해제** | 전체 Reset 대상 해제 |
| **AP Reset 실행** | 빨간색으로 지정된 AP 목록을 확인 후 일괄 reset |

**실행 절차:**
1. 테이블에서 Reset할 AP를 클릭합니다 (Shift 클릭으로 여러 개 동시 선택 가능).
2. **AP Reset 선택** 버튼을 클릭합니다 → 선택한 AP 행이 빨간색으로 변합니다.
3. 필요한 AP를 모두 빨간색으로 지정한 뒤 **AP Reset 실행** 버튼을 클릭합니다.
4. 대상 AP 목록이 표시된 확인 창에서 **예**를 클릭합니다.
5. 각 AP에 `reboot` 명령이 전달되고 성공/실패 결과가 표시됩니다.

> Reset 실행 기록은 `runtime/operations.log`에 저장됩니다.

### EFG Remote 탭

> `can_view_efg_tab` 권한이 있는 사용자만 접근 가능합니다.

**대시보드 조회 버튼**을 클릭하면 아래 정보를 한 번에 가져옵니다:

| 섹션 | 표시 정보 |
|------|----------|
| 시스템 정보 카드 | Hostname, Uptime, Kernel, CPU Load, Memory (사용/전체/여유 MB) |

| 트래픽 통계 테이블 | 인터페이스별 수신/송신 MB, 패킷 수, 오류 수 (/proc/net/dev) |

> 모든 테이블 헤더를 클릭하면 해당 컬럼 기준으로 정렬됩니다.

## 자주 묻는 질문

**Q. VPN 연결했는데 "교회 내부 네트워크에 접속할 수 없습니다" 메시지가 나옵니다.**  
A. VPN이 완전히 연결된 뒤 GUI를 실행하세요. VPN 연결 직후 몇 초 대기 후 재시작하면 해결됩니다.

---

**Q. 6자리 코드를 입력했는데 "코드가 올바르지 않습니다"가 나옵니다.**  
A. 코드는 30초마다 바뀝니다. 앱에서 새 코드로 바뀐 직후 입력하거나, PC 시간이 정확한지 확인하세요.  
Windows 시간 자동 동기화: 설정 → 시간 및 언어 → 지금 동기화.

---

**Q. 스마트폰을 교체했습니다. 어떻게 해야 하나요?**  
A. 관리자에게 TOTP 초기화를 요청하세요. 초기화 후 새 기기에서 QR 코드를 다시 스캔하면 됩니다.

---

**Q. Log 동기화 버튼이 비활성화되어 있습니다.**  
A. `can_sync_nas` 권한이 없는 계정입니다. 필요 시 관리자에게 권한 설정을 요청하세요.

---

**Q. AP Reset 버튼이 회색(비활성)으로 표시됩니다.**  
A. `can_reset_ap` 권한이 없는 계정입니다. AP 리셋이 필요하면 관리자에게 요청하세요.

---

**Q. AP Remote 조회 시 모든 AP가 "Auth failed" 오류가 납니다.**  
A. AP SSH 비밀번호가 틀렸거나 저장된 비밀번호가 변경되었습니다.  
AP Remote 탭 → **AP 비밀번호 삭제** 버튼 → 조회 재시도 시 새 비밀번호 입력.

---

**Q. AP Remote 조회 시 EFG 비밀번호를 물어봅니다.**  
A. AP 조회는 EFG를 통해 이루어집니다. EFG 비밀번호를 입력하면  
Windows Credential Manager에 저장할지 묻습니다. **예**를 선택하면 이후 자동 사용됩니다.

---

**Q. AP Reset 선택(빨간색)으로 지정했는데 조회 후에도 빨간색이 유지되나요?**  
A. 네, 조회해도 빨간색 Reset 대상 지정은 유지됩니다. **AP 선택 해제** 또는 **모두 해제** 버튼으로 해제하세요.

---

**Q. 개발자 모드가 동작하지 않습니다.**  
A. VPN 연결 상태에서 `사용자 관리` 탭 → **이 컴퓨터 개발자 등록**을 다시 실행하세요.  
Machine ID는 Windows MachineGuid 기반이므로 VPN 어댑터 변경의 영향을 받지 않습니다.

---

**Q. EFG Remote에서 인터페이스 테이블이 비어 있습니다.**  
A. EFG에서 `ip addr show` 명령이 지원되지 않을 수 있습니다.  
EFG 접속이 성공했는데도 비어 있다면 시스템 정보 카드의 Hostname이 표시되는지 확인하세요.

---

**Q. 처음 설치 후 `pip install` 중 오류가 납니다.**  
A. Python이 PATH에 등록되지 않았을 수 있습니다.  
`python --version`이 Microsoft Store 메시지를 보이면 Windows 설정에서 `App execution aliases`를  
열고 `python.exe`, `python3.exe` 별칭을 끄세요.

---

**Q. 같은 PC에서 여러 팀원이 사용할 수 있나요?**  
A. 각 팀원이 개별 Windows 계정으로 로그인하면 Windows Credential Manager가 계정별로 분리됩니다.  
같은 Windows 계정 공유 사용은 권장하지 않습니다.
