# NVC NetHub — 사용자 안내

이 문서는 팀원이 NVC NetHub를 설치하고 처음 로그인할 때 필요한 내용을 정리합니다.

## 사전 준비

| 항목 | 내용 |
|------|------|
| VPN | 교회 내부망 VPN 연결 필요 |
| GitHub 계정 | 관리자에게 전달한 개인 GitHub 계정 |
| GitHub 초대 | `newvisionchurch-it`와 `newvisionchurch/nvc_security` 초대 수락 |
| GitHub PAT | 본인 GitHub 계정에서 발급한 Personal Access Token |
| OTP 앱 | Google Authenticator 또는 Authy |

## GitHub PAT 발급

1. GitHub 로그인
2. Settings → Developer settings
3. Personal access tokens → Tokens classic
4. `Generate new token`
5. `repo` 권한 선택
6. 생성된 PAT를 안전하게 보관
7. PAT는 생성 후 한 번만 표시되므로 반드시 안전한 곳에 저장합니다

다른 사람의 PAT를 사용하지 말고 본인 GitHub 계정의 PAT를 사용합니다.

## 필수 소프트웨어

### Windows

| 소프트웨어 | 권장 버전 | 용도 | 필수 여부 |
|------------|-----------|------|-----------|
| Windows | 10 이상 | 운영체제 | 필수 |
| Python | 3.10 ~ 3.13 | 소스 실행 (exe 사용자는 불필요) | source 사용 시 필수 |

**Python 설치 (Windows — source 사용자만)**

공식 사이트에서 설치합니다. 설치 시 **"Add Python to PATH"** 체크박스를 반드시 선택합니다.

```powershell
winget install Python.Python.3.12
```

또는 [python.org](https://www.python.org/downloads/) 에서 직접 다운로드합니다.

설치 확인:

```powershell
python --version
```

```bash
brew install python@3.12

```

```bash
python3.12 --version
python3.12 -c "import tkinter; print('tkinter OK')"
```

## 설치

1. GitHub 로그인
2. `nvc_release` 저장소 접속
3. 배포받은 버전 폴더에서 패키지 다운로드
4. 압축 해제
5. VPN 연결
6. NVC NetHub 실행

**Windows 사용자**는 `windows/NVC_NetHub_[버전]_win_exe.zip`을 사용합니다.

**Windows 개발자**는 `windows/NVC_NetHub_[버전]_win_source.zip`을 사용합니다.

**macOS 사용자**는 `macOS/NVC_NetHub_[버전]_mac_source.zip`을 사용합니다.

### Windows 설치 방법

압축 해제 후 `NVC_NetHub.exe`를 실행합니다.

**Windows 개발자 (source)**

압축 해제 후 PowerShell에서 실행합니다.

```powershell
.\run.ps1
```

최초 실행 시 가상 환경과 의존성이 자동으로 설치됩니다 (약 1~2분 소요).
이후 실행부터는 바로 앱이 시작됩니다.

### macOS 설치 방법

위 [필수 소프트웨어 — macOS](#macos) 항목의 Homebrew·Python 설치를 먼저 완료합니다.

압축을 해제한 폴더에서 터미널을 열고 실행합니다.

```bash
bash run.sh
```

최초 실행 시 가상 환경과 의존성이 자동으로 설치됩니다 (약 1~2분 소요).
이후 실행부터는 바로 앱이 시작됩니다.

> **주의:** 터미널에서 `bash run.sh` 명령으로 실행해야 합니다.
> 파인더에서 직접 더블클릭으로 실행하는 방식은 지원하지 않습니다.

## 버전 업데이트

### 기존 설정 유지

기존 설정을 그대로 유지하려면 `NVC_NetHub.exe`만 덮어씁니다.

```
NVC_NetHub\
├── NVC_NetHub.exe  ← 새 버전으로 교체
└── config\         ← 그대로 유지
```

### 기본값으로 초기화

설정이 꼬이거나 새 버전의 기본값으로 새로 시작하려면 `config\app_config.json`을 삭제합니다.
다음 실행 시 기본값으로 자동 복원됩니다.

```
config\app_config.json  ← 삭제하면 다음 실행 시 기본값으로 자동 생성
```

### config 폴더 파일 안내

| 파일 | 설명 |
|------|------|
| `app_config.json` | 앱 설정 파일. 설정 탭에서 저장할 때 갱신됩니다. 삭제하면 다음 실행 시 기본값으로 자동 복원됩니다. |
| `ap_inventory.json` | AP 목록 데이터. 관리자가 관리하며 사용자가 수정하지 않습니다. |
| `ssh_targets.example.json` | SSH 설정 구조 참고용 샘플. 수정하지 않습니다. |

## 첫 실행

NVC NetHub를 실행하면 먼저 팀원 인증 화면이 표시됩니다.

```text
1단계 VPN 접속 인증
2단계 팀원 인증 (GitHub ID & PAT)
```

팀원 인증에서 본인 GitHub ID와 PAT를 입력합니다.
앱은 PAT의 GitHub ID, `nvc_security` 접근 권한, NetHub 사용자 등록 여부를 확인합니다.

팀원 인증이 끝나면 로그인 화면으로 이동합니다.
자동으로 넘어가지 않으므로 `로그인 화면 이동` 버튼을 클릭하거나 Enter를 누릅니다.
로그인 화면에서는 NetHub 계정 ID, 비밀번호, OTP를 입력합니다.

로그인한 NetHub 계정에 등록된 GitHub ID와 현재 PAT의 GitHub ID가 같아야 로그인할 수 있습니다.

### 최초 로그인 시 "접속 비밀번호 미등록" 메시지

**처음 로그인하는 경우, OTP 등록 후 아래 메시지가 표시될 수 있습니다.**

```text
접속 비밀번호가 아직 등록되지 않았습니다.
관리자에게 비밀번호 등록을 요청하세요.
```

이것은 오류가 아닙니다. 접속 비밀번호는 OTP 등록 이후에 관리자가 설정할 수 있습니다.
순서상 첫 로그인에서 반드시 거치는 단계입니다.

**처리 순서:**

1. 위 메시지가 표시되면 관리자에게 "비밀번호 등록 요청"을 전달합니다.
2. 관리자가 등록을 완료한 뒤 재로그인하면 정상 진입됩니다.

## 로그인 실패 시 확인

| 메시지 | 확인할 내용 |
|--------|-------------|
| PAT 없음 | PAT를 입력했는지 확인 |
| PAT 인증 실패 | PAT 만료, 권한, 복사 오류 확인 |
| nvc_security 로드 실패 | GitHub 초대를 수락했는지 관리자에게 확인 |
| 등록된 팀원 아님 | 관리자에게 NetHub 사용자 GitHub ID 등록 요청 |
| 개인 인증 실패 | 본인 GitHub 계정의 PAT를 사용했는지, 관리자에게 등록된 GitHub ID가 맞는지 확인 |
| OTP 오류 | 스마트폰 시간 자동 동기화 확인 |
| 접속 비밀번호가 아직 등록되지 않았습니다 | 최초 로그인 시 정상 — 관리자에게 비밀번호 등록 요청 후 재로그인 |

## 주의

- PAT와 비밀번호는 다른 사람과 공유하지 않습니다.
- 관리자 PAT 또는 다른 팀원 PAT를 사용하지 않습니다.
- GitHub 초대가 `Pending Invite` 상태이면 로그인할 수 없습니다.
- VPN 연결 후 실행합니다.
